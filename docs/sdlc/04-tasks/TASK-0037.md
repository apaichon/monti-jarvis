# TASK-0037: ChillPay Callback Receiver

## Outcome

Monti accepts ChillPay payment callbacks securely, persists events once, and acknowledges — ready for Sprint 9 checkout handlers.

## Acceptance Criteria

1. `POST /api/callbacks/chillpay` — no JWT; parses `application/x-www-form-urlencoded` into `ChillPayCallbackForm` (`OrderNo`, `Amount`, `TransactionId`, `PaymentStatus`, `CheckSum`, …).
2. `VerifyCallback` validates MD5 `CheckSum` (concat order per ChillPay V2 spec + `md5_key`); invalid checksum → `400`.
3. Inserts into `payment_callback_events`; duplicate `transaction_id` returns `200` without re-insert.
4. Misconfigured gateway → `503`.
5. Dev bypass: `PAYMENT_CALLBACK_DEV_BYPASS=true` accepts form without checksum (local only, documented).
6. Unit test: valid checksum fixture + duplicate idempotency + `PaymentStatus` mapping (`0` success, `1` pending, `2` failed).

## Verification

```bash
go test ./internal/payment/... -run Callback -count=1
# Form POST with computed CheckSum (see chillpay.VerifyCallback)
curl -X POST http://localhost:8091/api/callbacks/chillpay \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "TransactionId=999&Amount=10000&OrderNo=test-1&PaymentStatus=0&CheckSum=<computed>"
psql $DATABASE_URL -c "SELECT payment_status, transaction_id FROM callcenter.payment_callback_events ORDER BY received_at DESC LIMIT 5;"
```
