# TASK-0037: Stripe Webhook Receiver

## Outcome

Monti accepts Stripe webhooks securely, persists events once, and acknowledges — ready for Sprint 9 checkout handlers.

## Acceptance Criteria

1. `POST /api/webhooks/stripe` — no JWT; verifies `Stripe-Signature` with configured webhook secret (env or DB).
2. Parses `id` and `type` from event JSON; inserts into `payment_webhook_events`; duplicate `provider_event_id` returns `200` without re-insert.
3. Invalid/missing signature → `400`; misconfigured gateway → `503`.
4. Dev bypass: `PAYMENT_WEBHOOK_DEV_BYPASS=true` accepts raw JSON without signature (local only, documented).
5. Unit test: valid signature fixture + duplicate idempotency.

## Verification

```bash
go test ./internal/payment/... -run Webhook -count=1
# Stripe CLI
stripe listen --forward-to localhost:8091/api/webhooks/stripe
stripe trigger payment_intent.succeeded
psql $DATABASE_URL -c "SELECT event_type, provider_event_id FROM callcenter.payment_webhook_events ORDER BY received_at DESC LIMIT 5;"
```
