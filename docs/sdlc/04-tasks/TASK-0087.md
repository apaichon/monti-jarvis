# TASK-0087: Customer Identity Schema

## Outcome

Postgres `callcenter` tables for tenant customer records, CSV import summaries, domain rules, and customer-group membership.

## Acceptance criteria

1. Add `customers`, `customer_group_members`, `customer_import_jobs`, and `customer_domain_rules` with audit columns.
2. `customers` has tenant-scoped normalized email and `(tenant_id, source, external_id)` idempotency indexes.
3. Tier/group foreign keys cannot cross tenants; service validation supplements database FKs where composite keys are not used.
4. Customer deletion is modeled as `status=inactive`; no credential or password columns are added.
5. Schema initialization is idempotent and covered by store tests or SQL assertions.

## Verification

## Implementation evidence

- `internal/store/customers.go` adds all four tenant-scoped tables and indexes through idempotent ensure-schema.
- `internal/store/audit.go` registers every table for audit columns and update triggers.
- `internal/store/customers_test.go` covers identity/domain/source normalization and credential-metadata rejection.
