# TASK-0151: Tenant Theme editor

## Outcome

Tenant admins configure presets and color tokens with live preview, contrast warnings, draft save, publish, and reset in the tenant portal.

## Acceptance criteria

1. Route e.g. `/tenant/theme` (or Settings → Theme) reachable for active `tenant_admin`.
2. UI supports preset select (light / dark / branded), color inputs for each token, live preview panel approximating caller desk chrome.
3. Save draft and Publish actions call TASK-0149 APIs; success/error feedback.
4. Contrast failures show clear warnings; publish requires confirmation when warnings present.
5. Reset restores preset defaults into draft (and documents whether publish is separate).
6. EN labels primary; TH optional where settings pages already bilingual.
7. `npm run check` clean for tenant-web.

## Notes

App: `apps/tenant-web`. Reuse existing form/card patterns from `/tenant/settings` and `/tenant/embed`.
