# TASK-0141: Platform billing, quota, and AI usage API

## Outcome

Expose bounded platform-admin aggregate billing, quota, historical usage, and AI cost coverage through a stable read-only API.

## Acceptance criteria

1. `GET /api/platform/billing/usage` validates date range, tenant filter, limit, and offset.
2. Platform RBAC is enforced independently of the optional auth development bypass.
3. Response sections identify freshness, observed/estimated/unavailable AI usage, reconciliation state, and partial source failures.
4. Tenant rows contain only allowlisted aggregate and safe package fields.
5. ClickHouse, Postgres, Redis, and rate lookup failures never leak raw infrastructure details or become false zero values.

## Review — PASS

The read-only `/api/platform/billing/usage` endpoint applies the platform-admin guard, validates bounded date/filter/pagination inputs, joins paid-order, activity, AI projection, active-tenant, and current quota snapshots through allowlisted aggregate response types, and returns explicit freshness/reconciliation/measurement states. Required source failures return safe `usage_unavailable` responses; optional AI failure remains partial without infrastructure details. Daily quota availability is explicit rather than represented as a false zero.

Verification: parser, quota-gap, AI coverage, ClickHouse query, full Go test, server build, platform-admin checks/build, and `git diff --check` pass.
