# TASK-0120: Normalized Dependency Probes and Performance Snapshot

## Outcome

Provide a bounded, reusable server-side probe layer for the dependencies already used by Monti Jarvis.

## Acceptance criteria

1. Probe Postgres, Redis, MinIO, and ClickHouse using the existing clients, and report NATS, LiveKit, and Gemini from their existing enabled/configured state without adding a new infrastructure dependency.
2. Each result has a normalized status, checked timestamp, and non-negative latency when the probe completes.
3. Probe timeouts and provider errors are mapped to stable categories; credentials, URLs, and raw provider messages never enter the tenant response.
4. Probes run outside customer call, chat, quota, archive, and `/healthz` critical paths.
5. Unit tests cover configured, disabled, timeout, and failed dependency states.

## Implementation Evidence

- Added `internal/observability` with concurrent normalized probes, shared timeout, stable ordering, and redacted status aggregation.
- Added existing-client probes for Postgres, Redis, MinIO, ClickHouse, NATS, LiveKit, and Gemini without new infrastructure or persistence.
- Added unit coverage for operational, disabled, unavailable, stale analytics, and timeout behavior.

## Review - PASS

The implementation matches DES-0029: no raw provider errors enter the response, monitoring is request-scoped and outside call/archive paths, and the probe timeout is bounded by `MONITORING_PROBE_TIMEOUT`.
