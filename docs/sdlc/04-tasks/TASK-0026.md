# TASK-0026: Tenant Registration API

## Outcome

Public endpoint atomically creates tenant (`pending_kyc`), default brand, admin user, `tenant_admin` role, and `tenant_registrations` audit row.

## Acceptance Criteria

1. `POST /api/public/tenant/register` accepts `{ company_name, slug, admin_email, admin_password, admin_display_name }`.
2. Slug validated: `^[a-z0-9]([a-z0-9-]{1,30}[a-z0-9])?$`; stored lowercase; unique → else `409`.
3. Email unique across `users` → else `409`; password min 8 chars → else `400`.
4. Transaction creates: `tenants` (`pending_kyc`), `brands`, `users`, `user_roles` (`tenant_admin`), `tenant_registrations` (`status= submitted`).
5. Response `201`: `{ tenant_id, slug, registration_id, access_token, refresh_token, expires_in }`.
6. Unit tests for validation and collision paths in `internal/store` or `internal/tenantregister`.

## Verification

```bash
curl -s -X POST http://localhost:8091/api/public/tenant/register \
  -H 'Content-Type: application/json' \
  -d '{"company_name":"Acme Corp","slug":"acme","admin_email":"admin@acme.test","admin_password":"secret1234","admin_display_name":"Acme Admin"}'
```
