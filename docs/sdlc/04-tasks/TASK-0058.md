# TASK-0058: Quota Service

## Outcome

Package `internal/quota` loads effective entitlement rules and maintains Redis (and/or Postgres) usage counters with check / increment / release APIs used by enforcement.

## Acceptance Criteria

1. `quota.Service` depends on `entitlements.Service` + Redis; methods roughly:
   - `Snapshot(ctx, tenantID) (Usage, Limits, error)`
   - `Check(ctx, tenantID, dimension) error` — `ErrLimitExceeded` when over
   - `AcquireConcurrent(ctx, tenantID) (release func, error)`
   - `AddCallMinutes(ctx, tenantID, minutes int) error`
   - `CheckKMDocument(ctx, tenantID) error` — compare count vs `max_km_documents`
   - `CheckAIEmployees(ctx, tenantID, nextCount int) error`
   - `AllowRate(ctx, tenantID, bucket string) error` — rate limit
2. Dimensions map to rules-v1 keys; missing entitlement → documented default (deny or demo package).
3. Unit tests cover: under limit, at limit, over limit, concurrent release, monthly key rollover boundary (mock clock if needed).
4. Rate limiter is reusable (generalize register IP limiter or new package under `internal/quota` / `internal/ratelimit`).
5. No HTTP handlers in this task (→ TASK-0059/0060).
