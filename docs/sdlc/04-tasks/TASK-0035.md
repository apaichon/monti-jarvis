# TASK-0035: Payment Gateway Schema

## Outcome

Postgres stores platform payment gateway configuration and an idempotent webhook audit log for Stripe (and future providers).

## Acceptance Criteria

1. Table `payment_gateway_configs` ‚Äî singleton platform config: `provider`, `mode` (`test`|`live`), `publishable_key`, `secret_key_set` bool, `webhook_secret_set` bool, `status` (`inactive`|`active`), audit columns.
2. Table `payment_webhook_events` ‚Äî `provider`, `provider_event_id`, `event_type`, `payload_hash`, `received_at`, audit columns; unique `(provider, provider_event_id)`.
3. `ensureSchema` in `internal/store` runs idempotently on server start; no row required for server boot.
4. Documented in [03-er-diagram.md](../02-design/03-er-diagram.md) Sprint 8 delta.

## Verification

```bash
make restart
psql $DATABASE_URL -c "\d callcenter.payment_gateway_configs"
psql $DATABASE_URL -c "\d callcenter.payment_webhook_events"
go test ./internal/store/... -count=1 2>/dev/null || go test ./...
```
