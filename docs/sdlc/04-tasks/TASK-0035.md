# TASK-0035: Payment Gateway Schema

## Outcome

Postgres stores platform ChillPay gateway configuration and an idempotent callback audit log.

## Acceptance Criteria

1. Table `payment_gateway_configs` — singleton platform config: `provider`, `mode` (`test`|`live`), `merchant_code`, `api_key`, `md5_key_set` bool, `base_url`, `route_no`, `currency` (default `THB`), `callback_url`, `return_url`, `status` (`inactive`|`active`), audit columns.
2. Table `payment_callback_events` — `provider`, `transaction_id`, `order_no`, `payment_status`, `amount`, `payload_hash`, `received_at`, audit columns; unique `(provider, transaction_id)`.
3. `ensureSchema` in `internal/store` runs idempotently on server start; no row required for server boot.
4. Documented in [03-er-diagram.md](../02-design/03-er-diagram.md) Sprint 8 delta.

## Verification

```bash
make restart
psql $DATABASE_URL -c "\d callcenter.payment_gateway_configs"
psql $DATABASE_URL -c "\d callcenter.payment_callback_events"
go test ./internal/store/... -count=1 2>/dev/null || go test ./...
```
