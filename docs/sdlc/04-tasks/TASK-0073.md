# TASK-0073: Settings APIs + Enforce Caps

## Outcome

Tenant-admin REST APIs and voice path enforcement for daily/per-call minutes.

## Acceptance Criteria

1. Auth: `RequireTenantAdminActive` on all routes; tenant from JWT only.  
2. Endpoints:

   | Method | Path | Purpose |
   | --- | --- | --- |
   | GET | `/api/tenant/settings` | Settings (+ lazy create) |
   | PUT | `/api/tenant/settings` | Update locale, timezone, labels |
   | GET | `/api/tenant/usage` | Package limits + usage snapshot |
   | GET | `/api/tenant/call-limits` | Call caps |
   | PUT | `/api/tenant/call-limits` | Update caps (validate ≥ 0) |

3. Validation: locale ∈ {en,th}; timezone valid IANA (or allowlist common zones); call limits ≥ 0.  
4. Voice: on open, check daily remaining; track session start; on end / periodic, enforce per-call max (end call or deny further audio).  
5. Errors: clear codes e.g. `daily_call_limit`, `per_call_limit` (403/429).  
6. Unit tests for clamp/validation and Redis daily incr.  
7. S13 monthly quota still applies first.  

## Verification

```bash
TOKEN=…
curl -sS -H "Authorization: Bearer $TOKEN" http://localhost:8091/api/tenant/usage | jq .
curl -sS -X PUT -H "Authorization: Bearer $TOKEN" -H 'Content-Type: application/json' \
  -d '{"max_minutes_per_call":2,"max_call_minutes_per_day":10}' \
  http://localhost:8091/api/tenant/call-limits | jq .
```
