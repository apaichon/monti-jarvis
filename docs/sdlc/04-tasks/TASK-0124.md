# TASK-0124: Versioned mobile API and bootstrap

## Outcome

Expose the versioned mobile bootstrap and call-session HTTP facade with safe, typed response envelopes.

## Acceptance criteria

1. Bootstrap returns tenant policy, locale/timezone, assigned active avatars, default avatar, and authoritative limits.
2. Create accepts an optional assigned avatar and never trusts a body tenant selector.
3. Status, end, transcript, and rating operations return bounded mobile-safe data.
4. Idempotency and cross-tenant/customer ownership checks are enforced.
