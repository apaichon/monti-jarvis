# TASK-0023: DB-Backed Workforce Resolver

## Outcome

Customer portal and voice/chat paths receive tenant-specific avatar lists from Postgres instead of only the static Go catalog.

## Acceptance Criteria

1. `GET /api/workforce` resolves tenant via `auth.ResolveTenant` (header `X-Tenant-Id`, auth context, or demo default).
2. When tenant has active assignments, response `agents[]` shape matches current JSON (id, name, role, trait, color, voice, voice_provider_id, voice_id, image, greeting, flags); `voice*` from primary `ai_avatar_voices` row (lowest `priority` among `active`).
3. When tenant has zero assignments, fall back to `workforce.All()` (backward compatible).
4. `SystemPrompt` / `Resolve` continue to work for chat and voice with assigned avatar ids.
5. Regression: `go test ./internal/workforce/...` and existing server tests pass.

## Verification

```bash
curl -H "X-Tenant-Id: demo" http://localhost:8091/api/workforce | jq '.agents | length'
```
