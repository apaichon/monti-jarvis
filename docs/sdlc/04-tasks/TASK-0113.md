# TASK-0113: Post-Conversation Star Review Flow

## Outcome

Customers are invited by the AI to rate the completed conversation, with a clear 1-5 star control for chat and voice sessions.

## Acceptance criteria

1. After a normal conversation end, the AI asks the customer to rate the service from 1 to 5.
2. Voice and chat clients submit the selected rating through the satisfaction API and show confirmation state.
3. The customer UI uses star icons for the 1-5 rating control and remains usable on narrow screens.
4. If a call ends without a rating, show a review prompt/modal that allows the customer to enter the score without reopening the call.
5. Repeated submit, refresh, or delayed voice events remain idempotent and do not create multiple reviews.
6. Preserve call archive completion and ticket/session close behavior when the customer skips the review.

## Review - PASS

- Voice completion opens the star dialog after archive/session cleanup; chat completion exposes the same dialog through `Finish chat & rate`.
- The rating control uses five star buttons, supports `Not now`, and submits through the existing idempotent rating endpoint.
- Customer UI validation passed with `npm run check` and `npm run build`.
- Live browser/audio UAT remains tracked in `MT-0024`.
