# TASK-0150: CSS token runtime (customer + embed)

## Outcome

Map published theme tokens to CSS custom properties on the customer caller desk and embed iframe so brand colors apply without hard-coded Monti-only blues when a tenant has published a theme.

## Acceptance criteria

1. Documented CSS variable map (e.g. `--mj-primary` → UI chrome) implemented for customer-web and embed route.
2. Default **dark** preset renders within documented visual parity of pre-S39 palette when no custom theme published.
3. Public endpoint or existing bootstrap/resolve returns **published** tokens only (never draft).
4. Embed path: after public embed resolve (or theme resolve), apply tokens before paint of chrome (no FOUC worse than one frame when feasible).
5. Tenant B’s published theme does not affect Tenant A sessions.
6. Optional query/prop `theme` remains non-breaking when feature unset.

## Notes

Surfaces: `apps/customer-web`, `apps/customer-web/src/routes/embed`, public API. Framework SDKs may pass through `theme` only — no SDK package release required unless needed for types.
