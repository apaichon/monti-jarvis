# TASK-0112: Customer Satisfaction Review Schema and API

## Outcome

Store one tenant-scoped 1-5 satisfaction review for a completed conversation and expose a safe API for customer submission and tenant reporting.

## Acceptance criteria

1. Add an idempotent satisfaction review schema linked to tenant, call, conversation, avatar, channel, and completed timestamp.
2. Enforce a rating range of 1-5 and prevent duplicate reviews for the same call/conversation.
3. Accept customer review submission only for the originating tenant and completed conversation.
4. Return aggregate-safe review data without raw transcript, contact secrets, or cross-tenant metadata.
5. Add tests for validation, duplicate submission, missing call, and tenant isolation.

## Review - PASS

- Added tenant-scoped `conversation_ratings` metadata, range validation, unique `(tenant_id, call_id)` upsert, and audit fields.
- Rating submission resolves the completed conversation within the tenant scope before persisting.
- Aggregate store tests cover reviewed/unrated counts, distribution, average, and dimension buckets.
- Verification: `/usr/local/go/bin/go test ./...` and `/usr/local/go/bin/go build ./cmd/server` passed.
