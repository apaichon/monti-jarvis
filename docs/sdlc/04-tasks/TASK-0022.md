# TASK-0022: Tenant Avatar Assign / Revoke / List + Entitlement Cap

## Outcome

Platform admins assign avatars to tenants; assignment respects active entitlement `max_ai_employees`.

## Acceptance Criteria

1. `GET /api/platform/tenants/{tenant_id}/avatars` — assigned avatars with catalog metadata.
2. `POST` body `{ "avatar_id": "..." }` — upsert active assignment.
3. `DELETE /api/platform/tenants/{tenant_id}/avatars/{avatar_id}` — revoke/disable assignment.
4. Before assign: count active assignments vs `entitlements.GetEffective` → `rules.max_ai_employees`; `409` when at cap.
5. `404` for unknown tenant or avatar; audit columns on writes.

## Verification

```bash
curl -H "Authorization: Bearer $TOKEN" http://localhost:8091/api/platform/tenants/demo/avatars
```
