# TASK-0029: Tenant Registration UI

## Outcome

New SvelteKit app `apps/tenant-web` served at `/tenant/` with a registration form matching design wireframes T1–T3.

## Acceptance Criteria

1. `apps/tenant-web` builds via `make tenant-web`; Go serves static files at `/tenant/` (mirror platform-admin pattern).
2. `/tenant/register` form fields: company name, slug (auto-suggest from name), admin email, password, confirm password, display name.
3. Submit calls `POST /api/public/tenant/register`; shows field errors from API `400`/`409`.
4. Success page stores tokens (session/localStorage per platform-admin pattern) and shows “pending verification” message.
5. Link to future tenant login (`/tenant/login` stub OK) and back to customer portal `/`.
6. `make build` includes tenant-web; `healthz` reports `tenant_web` path when configured.

## Verification

```bash
make build && make restart
open http://localhost:8091/tenant/register
```
