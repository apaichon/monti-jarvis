# TASK-0041: Tenant Checkout APIs

## Outcome

Active tenants list packages and start ChillPay checkout; mock provider works in CI.

## Acceptance Criteria

1. `GET /api/tenant/packages` — active packages with limits summary + `price_cents`; `tenant_admin` + `active` tenant only.
2. `POST /api/tenant/checkout` body `{package_id}` — creates `pending` order, unique `order_no`, calls `ChillPayClient.InitPayment` (amount from `price_cents`, satang).
3. Mock provider returns local `payment_url` (`/tenant/billing/mock-pay?order=...`) that triggers fulfillment dev route.
4. `GET /api/tenant/orders/{id}` — order status; `404` for other tenant's order.
5. `409` when tenant not `active`, package not `active`, or gateway not configured.

## Verification

```bash
go test ./internal/store/... ./cmd/server/... -run Checkout -count=1
curl -H "Authorization: Bearer $TENANT_TOKEN" http://localhost:8091/api/tenant/packages
curl -X POST -H "Authorization: Bearer $TENANT_TOKEN" -H "Content-Type: application/json" \
  -d '{"package_id":"pkg-pro"}' http://localhost:8091/api/tenant/checkout
```
