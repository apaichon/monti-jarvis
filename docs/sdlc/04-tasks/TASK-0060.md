# TASK-0060: Platform Usage View

## Outcome

Platform admins read **usage vs limits** for any tenant via API and a portal panel.

## Acceptance Criteria

1. `GET /api/platform/tenants/{tenant_id}/usage` — `platform_admin` only; body includes limits (from entitlement), usage counters, package slug/status.
2. `tenant_admin` receives **403** on platform route; may later use entitlements/me (out of scope to expand tenant UI this sprint).
3. Platform admin UI: usage section on tenant detail **or** `/admin/usage?tenant_id=` — shows bars or table for concurrent, minutes (month), KM docs, AI employees.
4. Empty/no entitlement: UI shows clear empty state (not crash).
5. Client API helper in `apps/platform-admin-web`.
