# TASK-0028: Platform Tenant List API

## Outcome

Platform operators can list tenants with status filter — foundation for Sprint 7 KYC queue.

## Acceptance Criteria

1. `GET /api/platform/tenants` requires `platform_admin`; supports `?status=pending_kyc|active|suspended` and `?limit` / `?offset`.
2. Response includes `id`, `slug`, `name`, `status`, `created_at`, optional `registration_id`.
3. `403` for `tenant_admin`; `401` without token when auth enabled.
4. Minimal list row in platform admin UI (table link from `/admin` nav) — optional stretch; API is required.

## Verification

```bash
TOKEN=$(curl -s -X POST http://localhost:8091/api/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"platform@monti.local","password":"monti-platform"}' | jq -r .access_token)
curl -H "Authorization: Bearer $TOKEN" "http://localhost:8091/api/platform/tenants?status=pending_kyc"
```
