# TASK-0027: Registration Auth Integration

## Outcome

New registrants receive working JWT session; `GET /api/auth/me` reflects tenant; pending tenants have documented access boundaries.

## Acceptance Criteria

1. Registration handler uses existing `auth.Service` to issue access + refresh after user create (same rotation rules as login).
2. `GET /api/auth/me` returns `tenant_id`, `role=tenant_admin`, `email` for registered user.
3. `pending_kyc` tenants: login succeeds; document in spec which routes are blocked vs allowed (reads OK, no new commerce APIs).
4. NATS auth event `tenant.registered` published when bus enabled (optional stub log if NATS down).
5. `AUTH_DISABLED` does not bypass registration API (always writes to DB when Postgres up).

## Verification

```bash
# After register, use returned access_token
curl -H "Authorization: Bearer $TOKEN" http://localhost:8091/api/auth/me
```
