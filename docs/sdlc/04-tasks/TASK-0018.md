# TASK-0018: Platform Admin Portal — Login, Logout, Profile, Shell

## Outcome

SvelteKit app `apps/platform-admin-web` served at `/admin` with authenticated shell for `platform_admin`.

## Acceptance Criteria

1. `make platform-admin-web` builds static app; Go serves at `/admin/` via `internal/platformweb` (SPA fallback).
2. `/admin/login` — email/password form → `POST /api/auth/login` → store tokens in `sessionStorage` → redirect `/admin/packages`.
3. Logout button → `POST /api/auth/logout` (Bearer + refresh) → clear storage → `/admin/login`.
4. `/admin/profile` — `GET /api/auth/me` shows email, display_name, role, tenant_id (read-only).
5. Route guard: non-login `/admin/*` requires valid `platform_admin` token; 401 → login; `tenant_admin` → forbidden message.
6. Layout matches [05-ux-ui.md](../02-design/05-ux-ui.md) screens **P0** (login), **P1** (shell), **P2** (profile).
7. `make build` includes platform-admin-web; customer portal `/` unchanged.

## Verification

```bash
make platform-admin-web && make restart
# browser: http://localhost:8091/admin/login
```
