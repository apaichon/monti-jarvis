# TASK-0110: Tenant Ticket UI

## Outcome

Tenant admins have an operations screen to find, inspect, triage, and close tickets without leaving the tenant portal.

## Acceptance criteria

1. Add a Tickets navigation item and route under the tenant operations area.
2. Queue supports default recent/today view plus date, status, priority, category, avatar, and assignee filters.
3. Detail panel shows safe requester context, source conversation summary/link, priority, assignee, status, and event timeline.
4. Controls update status, priority, assignee, and internal notes with loading, error, and success states.
5. Desktop and mobile layouts keep queue/detail controls readable without overlapping content.
6. `npm run check` passes for `apps/tenant-web`.

## Verification

```bash
cd apps/tenant-web && npm run check && npm run build
```
