# TASK-0128: Audit event contract and HTTP emission

## Outcome

Capture redacted mutation outcomes with stable event IDs and actor/tenant/request context at the Go HTTP boundary.

## Acceptance criteria

1. Mutating API requests emit structured audit events without capturing request bodies or credentials.
2. Tenant and actor values resolve from authenticated context, trusted tenant routing, or explicit anonymous/system values.
3. Authorization and failure outcomes are represented without exposing another tenant's existence.
4. Event metadata is bounded and allowlisted.

## Verification

- Added `internal/audit.Event`, actor mapping, resource/action derivation, request ID handling, and mutation middleware.
- Added tests for stable context and body-free metadata.
