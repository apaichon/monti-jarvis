# TASK-0008: KM Ingest API — MinIO → Chunk → Embed → ClickHouse

## Outcome

Operators upload KM files via API; pipeline stores raw object in MinIO, chunks text, embeds via Gemini, and indexes vectors in ClickHouse.

## Acceptance Criteria

1. `POST /api/km/documents` accepts multipart upload (`.md`, `.txt`) with `scope` field and `X-Tenant-Id` header.
2. Raw file stored at MinIO prefix `km/{tenant_id}/{document_id}/original`.
3. Chunker splits on paragraphs/headings with max ~800 tokens per chunk; persists chunk rows in Postgres.
4. Embedding provider uses `GEMINI_API_KEY` (text-embedding model); failures mark document `index_failed`.
5. Successful run writes embeddings to ClickHouse and sets document status `indexed`.
6. Optional NATS publish `km.document.indexed` with document_id and chunk count.
7. Idempotent re-index bumps `km_version` and replaces prior ClickHouse rows for that document.
