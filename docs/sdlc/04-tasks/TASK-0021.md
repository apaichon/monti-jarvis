# TASK-0021: Avatar Catalog Store + Platform CRUD API

## Outcome

`internal/store/avatars.go` provides list/get/create/update/archive; HTTP handlers expose `/api/platform/avatars*` for `platform_admin`.

## Acceptance Criteria

1. `ListAvatars`, `GetAvatar`, `CreateAvatar`, `UpdateAvatar`, `ArchiveAvatar` with audit actor from context; voice profiles via `ListAvatarVoices`, `UpsertAvatarVoices` (replace-on-update).
2. `GET/POST /api/platform/avatars`, `GET/PUT/DELETE /api/platform/avatars/{id}` per API spec; detail/list include `voices[]`; create/update accept `voices[]` (`voice_provider_id`, `voice_id`, `voice`, `priority`, `status`).
3. Validation: required slug/name; slug lowercase unique; status `draft|active|archived`; at least one `active` voice profile when status is `active`.
4. `DELETE` soft-archives; `409` if active tenant assignments exist.
5. `RequirePlatformAdmin` on all routes; `go test` for handler/store unit tests where practical.

## Verification

```bash
TOKEN=$(curl -sS -X POST .../api/auth/login -d '{"email":"platform@monti.local","password":"monti-platform"}' | jq -r .access_token)
curl -H "Authorization: Bearer $TOKEN" http://localhost:8091/api/platform/avatars
```
