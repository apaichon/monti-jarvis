# TASK-0092: Customer Auth Schema

## Outcome

Postgres `callcenter` tables and indexes for customer OTP identities, OTP challenges, customer sessions, tenant auth settings, and audit-safe auth events.

## Acceptance criteria

1. Add customer auth identity records linked to SPRINT-019 `customers` and scoped by `tenant_id`.
2. Add OTP challenge and customer session storage with expiry, revocation, attempts, and audit columns.
3. Add tenant customer-auth settings for enabled/disabled state and domain enforcement mode.
4. Enforce tenant-scoped uniqueness for customer email auth identity without cross-tenant leakage.
5. Store OTP hashes only; no plaintext OTP, tokens, or secrets appear in logs, audit rows, or API responses.

## Verification

```bash
go test ./internal/store ./cmd/server
```

## Implementation notes

- Added `tenant_customer_auth_settings`, `customer_auth_identities`, `customer_otp_challenges`, `customer_sessions`, and `customer_auth_events`.
- OTP challenges store only HMAC hashes; refresh tokens are stored as hashes.
- Customer auth settings are tenant-scoped and audited.
