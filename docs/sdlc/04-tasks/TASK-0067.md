# TASK-0067: Delete Cascade + Route Shell

## Outcome

Safe delete of one knowledge document across Postgres, MinIO, and ClickHouse; **`km_gaps` table** for missing-KM FAQ backlog; register tenant KM HTTP handlers (impl may be thin until TASK-0068).

## Acceptance Criteria

1. `internal/km` (or store) exposes **DeleteDocument(tenantID, documentID)** that:
   - Verifies document belongs to tenant
   - Deletes chunks + document rows in Postgres
   - Deletes MinIO object when present
   - Deletes ClickHouse `km_embeddings` rows for that document_id / tenant
2. Unit or integration test: delete unknown id → not found; wrong tenant → not found/forbidden.
3. **`callcenter.km_gaps`** ensure-schema on startup with columns per DES-0018 / ER (question, hash, status, occurrence_count, audit…); unique `(tenant_id, agent_id, question_hash)`.
4. Chat path with `missing_km` calls **RecordKMGap** (upsert); ClickHouse `qa_events` still written.
5. `GET /api/tenant/km/gaps` + `PATCH /api/tenant/km/gaps/{id}` behind tenant admin guard.
6. `cmd/server` registers other `/api/tenant/km/*` routes behind `RequireTenantAdminActive` (handlers may 501 until 0068 if stubbed — prefer full wiring with 0068 same PR stack).
7. No regression to public customer chat success path.
8. Document any new env vars (none expected).
