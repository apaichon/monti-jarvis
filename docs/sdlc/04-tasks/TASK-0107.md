# TASK-0107: Ticket and Ticket-Event Schema

## Outcome

Postgres stores tenant-scoped tickets and an append-only event timeline with audit columns and validated lifecycle values.

## Acceptance criteria

1. Add `tickets` and `ticket_events` to the `callcenter` schema with tenant, conversation/call, customer, avatar, priority, status, assignment, and audit fields.
2. Add indexes for tenant queue queries, status/priority filtering, and ticket event chronology.
3. Enforce valid status, priority, category, and event-type values at the database or store boundary.
4. Add idempotency support for customer ticket creation without storing raw transcript or contact secrets in keys.
5. Schema bootstrap/migration is idempotent and covered by Go tests.

## Verification

```bash
go test ./internal/store/... ./internal/tickets/...
```
