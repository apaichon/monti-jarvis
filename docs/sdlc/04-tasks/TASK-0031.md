# TASK-0031: Platform KYC Review APIs

## Outcome

`platform_admin` can fetch a tenant's KYC package and record approve/reject decisions via REST.

## Acceptance Criteria

1. `GET /api/platform/tenants/{tenant_id}/kyc` — tenant + registration summary + `tenant_kyc_profiles` + photo/doc asset URLs.
2. `POST /api/platform/tenants/{tenant_id}/kyc/approve` — `200` on success; `409` if not `pending_kyc` or KYC not `submitted`; `404` unknown tenant.
3. `POST /api/platform/tenants/{tenant_id}/kyc/reject` — body `{ "reason": "..." }` required; `200` on success; same guards as approve.
4. `GET /api/platform/tenants?kyc_status=submitted` filters tenants with submitted KYC profiles.
5. All routes require `platform_admin`; `401`/`403` as per auth middleware.
6. Contract documented in [04-api-spec.md](../02-design/04-api-spec.md) and [12-kyc-tenant-spec.md](../02-design/12-kyc-tenant-spec.md).

## Verification

```bash
make test
TOKEN=$(curl -s -X POST http://localhost:8091/api/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"platform@monti.local","password":"monti-platform"}' | jq -r .access_token)
curl -H "Authorization: Bearer $TOKEN" http://localhost:8091/api/platform/tenants/{tenant_id}/kyc
curl -X POST -H "Authorization: Bearer $TOKEN" http://localhost:8091/api/platform/tenants/{tenant_id}/kyc/approve
```
