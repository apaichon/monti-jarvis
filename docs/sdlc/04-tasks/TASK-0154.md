# TASK-0154: Session expired handling

## Outcome

When the tenant session expires (or access token is invalid and refresh fails), the tenant UI always clears local session and routes to login with `reason=session_expired` and a safe `next` path; after re-login, user returns to the intended page.

## Acceptance criteria

1. Authenticated `apiFetch` 401 with prior token → single `handleUnauthorized` path; no stuck spinners.
2. Login page shows clear copy for `reason=session_expired`.
3. `next` query is validated (same-origin path only) and honored post-login.
4. No redirect loop when already on login.
5. Optional: customer portal same pattern if sharing helpers (document if out of scope for tenant-only).

## Notes

Touches: `apps/tenant-web/src/lib/api/http.ts`, `login/+page.svelte`, any direct `fetch` that bypasses `apiFetch` (preview/tax/operations — align them).
