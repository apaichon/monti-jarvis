# TASK-0125: Mobile voice WebSocket and lifecycle

## Outcome

Provide a mobile WebSocket entry point that reuses the existing quota-aware voice relay and validates the call session first.

## Acceptance criteria

1. Only the owning tenant/customer can connect to an active call.
2. Avatar, locale, quota, per-call cap, and rate-limit policy remain server-side.
3. Invalid call, expired session, and oversized-frame errors are safe and provider-independent.
4. Reconnect is possible while the call remains active and end-call remains explicit over HTTP.

## Verification

- Added `/ws/mobile/v1/calls/{call_id}` ownership validation and delegation to the existing quota-aware voice relay.
- Authenticated WebSocket clients may use the access token query handoff required by native/browser WebSocket adapters.
