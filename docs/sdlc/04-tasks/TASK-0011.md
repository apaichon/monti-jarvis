# TASK-0011: JWT Auth Service + /api/auth/* Endpoints

## Outcome

Package `internal/auth` issues and validates JWTs; HTTP handlers expose login, refresh, logout, and me.

## Acceptance Criteria

1. `JWT_SECRET` and token TTLs in `infra/.env.example` (`JWT_ACCESS_TTL`, `JWT_REFRESH_TTL`).
2. `POST /api/auth/login` — email + password → `{access_token, refresh_token, expires_in, user}`.
3. `POST /api/auth/refresh` — valid refresh token → new access + rotated refresh.
4. `POST /api/auth/logout` — invalidates refresh token server-side.
5. `GET /api/auth/me` — Bearer access token → user profile + role + tenant_id.
6. Passwords stored bcrypt; failed login returns generic `401` (no user enumeration).
7. Unit tests: token round-trip, expired token rejection, wrong password.

## Verification

```bash
go test ./internal/auth/...
curl -X POST http://localhost:8091/api/auth/login -d '{"email":"admin@demo.local","password":"..."}'
```
