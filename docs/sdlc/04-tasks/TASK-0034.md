# TASK-0034: Platform Admin KYC Review UI

## Outcome

Platform operators review KYC evidence in the admin portal and approve or reject with standardized feedback dialogs.

## Acceptance Criteria

1. `/admin/tenants` table row links to `/admin/tenants/{id}/kyc` (P12 wireframe).
2. Review page shows: company, slug, admin email, contact fields, photo preview, document list with download links.
3. **Approve** and **Reject** buttons; reject prompts for reason (modal or inline form).
4. Success/error via existing `feedback` dialog (not inline banners).
5. After approve, list filter `pending_kyc` no longer shows tenant; optional link to entitlement/avatars assign.
6. `make platform-admin-web` included in `make build`.

## Verification

```bash
make build
open http://localhost:8091/admin/tenants?status=pending_kyc
# Click tenant → review → approve
```
