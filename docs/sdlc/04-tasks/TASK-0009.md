# TASK-0009: Scoped RAG in Chat/Voice Orchestrator + UI Citations

## Outcome

Chat and voice turns retrieve scoped KM chunks before calling Gemini; Caller Desk shows citation chips; missing matches log `qa_events`.

## Acceptance Criteria

1. `/api/chat` and voice session embed the user turn, call `SearchScoped`, and inject top-k chunks into the system prompt.
2. Scope derived from active topic tab (`general` / `billing` / `technical`) and selected `agent_id`.
3. API response includes `sources[]` with document filename and chunk excerpt; portal renders chips under assistant bubbles.
4. When no chunk scores above configured threshold, assistant uses safe fallback (“I don’t have that in our knowledge base”) and writes `qa_events` with `event_type=missing_km`.
5. Call turn persistence stores `source_chunk_ids` JSON when session exists.
6. Regression: Sprint 1 voice/text flows still work when ClickHouse is offline (degraded mode, warning in logs).
