# TASK-0043: Tenant Billing UI

## Outcome

Tenants browse packages and start checkout from `/tenant/billing`.

## Acceptance Criteria

1. Route `/tenant/billing` — package cards, current entitlement from `GET /api/entitlements/me`, Buy button per package.
2. Buy calls `POST /api/tenant/checkout` → `window.location` to `payment_url`.
3. Route `/tenant/billing/return` — poll order or show success/pending/failed from query params.
4. Nav link in tenant shell; feedback dialog on errors.
5. `CHILLPAY_RETURN_URL` points to this return route.

## Verification

```bash
make tenant-web && make restart
open http://localhost:8091/tenant/billing
```
