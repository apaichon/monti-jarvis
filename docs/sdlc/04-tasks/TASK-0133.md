# TASK-0133: Platform system performance API

## Outcome

Expose the bounded cross-tenant snapshot to platform administrators through a protected, filterable REST endpoint.

## Acceptance criteria

1. `GET /api/platform/system-performance` supports exact tenant filtering, derived status filtering, and bounded pagination.
2. Non-platform callers receive the existing safe authorization response.
3. Response fields are allowlisted and status/error mapping never leaks infrastructure details.
4. Timeout and storage failures return `monitoring_unavailable` without raw errors.

## Verification

- Added route wiring, query validation, status derivation, pagination, redacted response types, and unit coverage in `cmd/server/platform_monitoring_test.go`.
- Verified `401` unauthenticated behavior, authenticated `200` response, and `400 validation_error` for an oversized limit.
