# TASK-0057: Quota Redis Keys, Env, Infra Status

## Outcome

Document and wire env/config for quota + rate limit; expose health on `/api/infra`; optional Postgres `quota_usage_events` (or skip if Redis-only MVP) for audit.

## Acceptance Criteria

1. Env documented in `infra/.env.dev.example` and LOCAL-DEV:
   - `QUOTA_ENABLED` (default true when Redis up)
   - `QUOTA_FAIL_OPEN` (default true)
   - `RATE_LIMIT_ENABLED`, `RATE_LIMIT_CHAT_PER_MIN` (sensible defaults)
   - `QUOTA_MONTH_TZ` or fixed UTC
2. Redis key conventions documented in [16-quota-rate-limit-spec.md](../02-design/16-quota-rate-limit-spec.md):
   - `monti_jarvis:quota:{tenant_id}:concurrent`
   - `monti_jarvis:quota:{tenant_id}:minutes:{YYYYMM}`
   - `monti_jarvis:quota:{tenant_id}:km_docs` (or derive from Postgres count)
   - `monti_jarvis:rl:{tenant_id}:chat:{window}`
3. `GET /api/infra` includes `quota: ok|disabled|degraded` and `rate_limit: ok|disabled`.
4. If Postgres audit table is chosen: migration/ensure-schema with audit columns; otherwise AC notes Redis-only and no DDL.
5. `make infra-check` still green.

## Verification

```bash
grep -E 'QUOTA_|RATE_LIMIT_' infra/.env.dev.example
curl -s http://localhost:8091/api/infra | jq '.quota, .rate_limit'
```

## Done notes (2026-07-11)

- Env: `QUOTA_*`, `RATE_LIMIT_*` in `internal/env`, `infra/.env.dev.example`, LOCAL-DEV
- Redis-only MVP (no `quota_usage_events` DDL)
- `/api/infra` → `quota` + `rate_limit` (`ok|disabled|degraded`)
- Month windows fixed **UTC**
