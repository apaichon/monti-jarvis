# TASK-0072: Settings & Limits Schema

## Outcome

Postgres tables for tenant settings and call-time caps; Redis key design for daily minutes; ensure-schema on startup.

## Acceptance Criteria

1. Table `callcenter.tenant_settings`:
   - `tenant_id` text PK FK → tenants  
   - `locale` text default `en` (`en`|`th`)  
   - `timezone` text default `Asia/Bangkok`  
   - `display_name` text nullable (workspace label)  
   - `ai_reply_locale` text default `` (empty = follow caller / locale)  
   - `user_tier_label` text default `` (scaffold)  
   - `user_group_label` text default `` (scaffold)  
   - audit columns  
2. Table `callcenter.tenant_call_limits`:
   - `tenant_id` text PK FK → tenants  
   - `max_minutes_per_call` int NOT NULL DEFAULT 0  (0 = unset)  
   - `max_call_minutes_per_day` int NOT NULL DEFAULT 0  
   - audit columns  
3. Lazy create defaults on first GET (like embed config).  
4. Redis keys (prefix `monti_jarvis:`):
   - `call_daily:{tenant}:{YYYYMMDD}` — seconds or minutes used today (UTC or tenant TZ — document choice; prefer **tenant timezone** for day boundary)  
   - Session timer remains in-process / S13 monthly keys unchanged  
5. Document env: none new required.
