# TASK-0040: Payment Orders Schema

## Outcome

Postgres tracks tenant checkout orders linked to ChillPay `OrderNo` and package catalog.

## Acceptance Criteria

1. Table `payment_orders`: `id`, `tenant_id` FK, `package_id` FK, `order_no` UK, `amount_cents`, `currency`, `status` (`pending`|`paid`|`failed`|`cancelled`), `provider`, `transaction_id`, `payment_url`, `paid_at`, audit columns.
2. Index on `(tenant_id, status)` and unique `order_no`.
3. `ensureSchema` idempotent; documented in ER Sprint 9 delta.
