# TASK-0030: KYC Review Schema

## Outcome

Postgres supports platform KYC approve/reject with reviewer audit columns; existing Sprint 6 rows migrate safely.

## Acceptance Criteria

1. `tenant_kyc_profiles.status` CHECK includes `draft` | `submitted` | `approved` | `rejected` (idempotent `ALTER`).
2. Columns added: `reviewed_at timestamptz`, `reviewed_by text`, `rejection_reason text` (default empty).
3. `tenant_registrations` gains `reviewed_at`, `reviewed_by`, `rejection_reason` (nullable) on approve/reject.
4. `ensureSchema` runs on server start; `demo` seed tenant unchanged (`active`).
5. Documented in [03-er-diagram.md](../02-design/03-er-diagram.md) Sprint 7 delta.
