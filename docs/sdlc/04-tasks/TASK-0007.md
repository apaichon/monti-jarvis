# TASK-0007: Postgres KM Documents, Chunks, and Scope Model

## Outcome

Postgres tables under `callcenter` schema store KM metadata and scope rules; demo tenant seeded with sample documents.

## Acceptance Criteria

1. Migrations add `knowledge_documents`, `knowledge_chunks`, `km_scope_assignments` tables with `tenant_id` column.
2. Document record tracks MinIO object key, filename, mime, status (`uploaded` → `indexed`), `km_version`, and `km_scopes` array.
3. Chunk record links to document, stores char offsets, content preview, and ClickHouse `chunk_id`.
4. Scope resolver maps conversation context (`topic` tab + `agent_id`) to allowed `km_scope` values for `demo` tenant.
5. `infra-init.sh` seeds at least two demo docs: one `general`, one `billing` (SQL or bundled Markdown).
6. `GET /api/km/documents` lists documents for `X-Tenant-Id: demo` (read-only dev API).

## Verification

```bash
make infra-init
psql "$POSTGRES_URL" -c "SELECT id, filename, km_scopes FROM callcenter.knowledge_documents"
go test ./internal/km/... ./internal/scope/...
```
