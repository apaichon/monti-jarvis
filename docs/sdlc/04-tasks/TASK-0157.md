# TASK-0157: Document scope on tenant KM

## Outcome

Tenants can assign a **scope** to KM documents on create/update, list/filter by scope, and retrieval respects scope allowlists so documents are not used outside intended agent/scope boundaries.

## Acceptance criteria

1. Upload / edit document UI includes scope selector (from existing tenant scopes).
2. API persists `scope_id` (or equivalent) on document metadata; list returns it.
3. Filter list by scope works in tenant KM UI.
4. Chat/RAG path does not retrieve docs outside the agent’s allowed scopes (align with S15 scope rules); unit or integration test covers isolation.
5. Existing documents with empty scope keep documented default (tenant-wide or “unscoped” policy in DES-0038 — prefer non-breaking default).

## Notes

Touches: `internal/store` KM, tenant KM handlers, `apps/tenant-web` KM routes, rag/scope filter. Confirm current schema before migration.
