# TASK-0001: AI Avatar Workforce + Inbound Q&A (Prototype → Roadmap Sprint 21)

## Outcome

Monti Jarvis lets inbound callers select an AI avatar agent (Ava, Max, Luna, Neo) and converse by text or voice to get answers to their questions.

## Acceptance Criteria

1. `GET /api/workforce` returns four agents with id, name, role, trait, color, voice, and greeting.
2. UI displays agent cards; selecting one updates the active avatar, halo color, and greeting.
3. `POST /api/chat` accepts `agent_id`, `topic`, `message`, and returns a role-aware reply using per-agent system prompts.
4. `GET /ws/voice?agent=<id>` opens a Gemini Live session with the agent's voice and system prompt.
5. Call timer, transcript stream, and text fallback work in the embedded UI.
6. Infra uses isolated `monti_jarvis` DB, Redis DB 4, and MinIO bucket `monti-jarvis`.
7. `go test ./...` passes; README documents run steps and planned-later features.
