# TASK-0114: Tenant Satisfaction Statistics

## Outcome

Tenant users can review aggregate AI satisfaction results for their own conversations and compare trends over time.

## Acceptance criteria

1. Add tenant-scoped statistics endpoints with start/end date filters and a default today filter.
2. Show total completed conversations, review count, average rating, rating distribution, and review completion rate.
3. Support breakdowns by avatar and channel without exposing individual customer identity or transcript content.
4. Add a tenant dashboard view with loading, empty, error, and date-filter states.
5. Ensure all queries enforce tenant scope and add API tests for cross-tenant access.
6. Keep dashboard values consistent with the underlying satisfaction review records.

## Review - PASS

- Added tenant-admin statistics API with default-today behavior, date validation, avatar filter, and chat/voice filter.
- Added tenant dashboard KPI, distribution, avatar/channel breakdown, loading, empty, and error states.
- Queries aggregate archived conversation records and ratings under the authenticated tenant scope without transcript/customer fields.
- Verification: tenant `npm run check` and `npm run build` passed.
