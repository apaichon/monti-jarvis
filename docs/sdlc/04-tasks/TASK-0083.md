# TASK-0083: Tiers APIs

## Outcome

Tenant-admin REST for customer tiers and groups.

## Acceptance Criteria

1. Auth: `RequireTenantAdminActive`; tenant from JWT only.  
2. Endpoints:

   | Method | Path |
   | --- | --- |
   | GET/POST | `/api/tenant/tiers` |
   | GET/PUT/DELETE | `/api/tenant/tiers/{id}` |
   | GET/POST | `/api/tenant/groups` |
   | GET/PUT/DELETE | `/api/tenant/groups/{id}` |

3. Validation: slug URL-safe unique per tenant; caps ≥ 0; locale empty|en|th.  
4. DELETE hard-delete OK while no customer FK (document).  
5. Unit tests for slug normalize / clamp.  
6. Errors: 404 cross-tenant or missing; 400 invalid body.

## Verification

```bash
TOKEN=…
curl -sS -H "Authorization: Bearer $TOKEN" http://localhost:8091/api/tenant/tiers | jq .
curl -sS -X POST -H "Authorization: Bearer $TOKEN" -H 'Content-Type: application/json' \
  -d '{"name":"VIP","slug":"vip","priority":10}' \
  http://localhost:8091/api/tenant/tiers | jq .
```
