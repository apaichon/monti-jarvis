# TASK-0033: KYC Decision Email

## Outcome

Tenant admin receives an email when platform approves or rejects their KYC package.

## Acceptance Criteria

1. On approve: send HTML email to `tenant_registrations.admin_email` with workspace name and "account active" message.
2. On reject: send email including `rejection_reason` and link to `/tenant/backoffice`.
3. Uses existing `internal/resend` client; no-op with log line when `RESEND_API_KEY` unset (dev friendly).
4. Email failure does not roll back DB transaction (log + continue).
5. `FROM` address reads from env (`RESEND_FROM_EMAIL` or project convention).

## Verification

```bash
# With RESEND_API_KEY in infra/.env.dev — approve tenant and check Resend dashboard / inbox
make restart
```
