# TASK-0109: AI Human-Follow-Up Offer

## Outcome

The conversation runtime can surface a structured human-follow-up offer and publish ticket lifecycle events only after customer confirmation.

## Acceptance criteria

1. Define a `ticket_offer` conversation event with bounded subject/category/reason fields and no raw secrets.
2. Trigger the offer from an explicit customer request for human help or an approved escalation signal; do not create a ticket from an offer alone.
3. Voice/chat clients receive the offer and can submit an explicit confirmation with an idempotency key.
4. Publish `ticket.created` and `ticket.updated` events with tenant, ticket, source conversation, and status metadata.
5. Preserve the existing conversation and archive flow when the customer declines or ignores the offer.
6. Add tests for offer, confirm, decline, duplicate confirm, and no-auth/required-auth policy behavior.
