# TASK-0155: First-login menu paint

## Outcome

After a successful tenant login (especially first visit / empty storage), the tenant shell and full navigation render immediately without requiring a manual page refresh.

## Acceptance criteria

1. Clear site data → open `/tenant/login` → login → land on default page with **all** nav groups visible.
2. Layout does not depend on a second navigation or hard reload to read session.
3. Session store / `hasRegistrationSession` (or equivalent) is reactive across login → layout.
4. Regression: refresh while logged in still shows menu; logout clears shell.

## Notes

Likely root causes: layout gated on non-reactive session read, or login `goto` before session write completes. Fix at session store + layout subscription.
