# TASK-0015: Package Catalog Service + Platform CRUD API

## Outcome

Package `internal/packages` exposes catalog CRUD; HTTP handlers under `/api/platform/packages*` require `platform_admin`.

## Acceptance Criteria

1. `GET /api/platform/packages` lists packages with optional `?status=` filter; includes limits.
2. `GET /api/platform/rule-schemas` lists active/deprecated schema versions with `fields` catalog.
3. `POST /api/platform/packages` creates package + `rules` jsonb; validates against `rules_schema_id` and schema `fields`.
4. `GET /api/platform/packages/{id}` returns package + `rules_schema_id` + `rules`; `404` when missing.
5. `PUT /api/platform/packages/{id}` updates metadata and/or `rules`; archived packages reject destructive rule changes when active entitlements exist.
6. `DELETE /api/platform/packages/{id}` soft-archives (`status=archived`); `409` if active tenant entitlements reference package.
7. `tenant_admin` and unauthenticated callers receive `403`/`401` on all platform package routes.
8. Unit tests: jsonb rules validation (unknown keys, types), archive guard, RBAC middleware.
