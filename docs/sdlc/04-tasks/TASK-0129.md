# TASK-0129: Local audit spool and transfer worker

## Outcome

Persist audit events in instance-local JSONL files and transfer closed files on a configurable Go routine interval.

## Acceptance criteria

1. Files use the `audit_log_YYYYMMDD-HH-MM-SS.jsonl` naming contract.
2. Closed files are transferred oldest-first and failed files remain available for retry.
3. Transfer acknowledgements are atomic and retention cleanup cannot delete unacknowledged files.
4. Shutdown closes the active file and performs a bounded final transfer.

## Verification

- Added `internal/audit.Writer` with local append, rotation, acknowledgement markers, retry scanning, and retention cleanup.
- Added `AUDIT_LOG_*` environment configuration and local example defaults.
- Added failure and acknowledgement tests.
