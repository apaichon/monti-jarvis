# TASK-0017: Entitlement Resolver + Redis Read-Through Cache

## Outcome

Package `internal/entitlements` resolves effective limits for a tenant; Redis caches results and invalidates on entitlement writes.

## Acceptance Criteria

1. `entitlements.Resolve(ctx, tenantID)` returns effective `rules` jsonb + `rules_schema_id` from active entitlement + `package_limits`; documented fallback when no entitlement (dev: permissive defaults or explicit `nil` — match packages-spec).
2. Redis key `monti_jarvis:entitlement:{tenant_id}` populated on cache miss when `ENTITLEMENT_CACHE_ENABLED=true`.
3. Assign, revoke, suspend, and package update affecting active tenants call cache invalidation (`DEL` key).
4. `GET /api/infra` includes `entitlement_cache: ok|disabled` when implemented alongside existing infra flags.
5. Unit tests: cache hit/miss, invalidation, TTL env `ENTITLEMENT_CACHE_TTL`.
6. Resolver callable from server layer for future Sprint 13 quota checks (no enforcement wired this sprint).
