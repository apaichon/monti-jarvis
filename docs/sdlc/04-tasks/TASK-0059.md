# TASK-0059: Path Enforcement

## Outcome

Wire `internal/quota` into live request paths so package limits are enforced end-to-end.

## Acceptance Criteria

1. **`POST /api/chat`**: rate limit; if RAG path, honor `rag_enabled`; return `429`/`403` with JSON `{ "error", "code" }` (`quota_exceeded` | `rate_limited` | `feature_disabled`).
2. **`GET /ws/voice`** (or session start): check `voice_enabled` + acquire concurrent slot; release on disconnect; reject with clear close/HTTP error when over `max_concurrent_calls`.
3. **KM ingest** (`POST /api/km/...`): reject when document count ≥ `max_km_documents`.
4. **Avatar assign** (platform assign API): reject when assigned count would exceed `max_ai_employees`.
5. Monthly minutes: increment on voice session end (best-effort); further sessions blocked when month total ≥ limit.
6. When `QUOTA_ENABLED=false` or fail-open + Redis error: paths succeed; log warning.
7. Integration or handler tests for at least one path (chat or KM).
