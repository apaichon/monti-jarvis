# TASK-0149: Theme schema and APIs (branding + colors)

## Outcome

Persist per-tenant draft/published **branding** (name, logo, subtitle) and **color tokens** with validation, logo upload, contrast helper, and tenant-admin APIs.

## Acceptance criteria

1. `tenant_themes` stores `draft_branding` / `published_branding` and `draft_tokens` / `published_tokens` with audit columns.
2. Branding validation: brand_name length, subtitle length, logo_url https/path rules.
3. Token validation for all required keys in DES-0037 (including `primary_text`, `surface_elevated`).
4. `POST /api/tenant/theme/logo` stores object under MinIO `theme/{tenant_id}/` and returns URL.
5. GET/PUT draft, POST publish (contrast confirm), POST reset work for active tenant_admin only.
6. Unit tests: validation, contrast, isolation, publish copies branding+tokens.

## Notes

See DES-0037 §§4–7. Screenshot header fields are first-class, not color-only.
