# TASK-0149: Theme schema and APIs

## Outcome

Persist per-tenant draft and published color token documents with presets, validation, contrast helper, and tenant-admin CRUD/publish/reset endpoints.

## Acceptance criteria

1. Migration/ensure-schema creates `callcenter.tenant_themes` (or equivalent per DES-0037) with audit columns and unique `tenant_id`.
2. Token payload validates: required keys, `#RRGGBB` (or documented formats), rejects unknown keys / empty publish.
3. Contrast helper returns pass/fail + ratio for text-on-surface and primary-on-surface (and documented extras).
4. Endpoints (names per DES): get theme, put draft, post publish, post reset; active `tenant_admin` only; tenant isolation enforced.
5. Publish copies draft → published and records `published_at` / `updated_by`.
6. Unit tests cover validation, contrast, and isolation denials.

## Notes

Touches: `internal/store`, `cmd/server` routes, optional Redis cache key `monti_jarvis:theme:{tenant_id}` if used (fail-open to Postgres).
