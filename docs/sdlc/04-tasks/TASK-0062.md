# TASK-0062: Embed Config Schema

## Outcome

Postgres table for per-tenant embed settings; idempotent ensure-schema; optional demo seed.

## Acceptance Criteria

1. Table `callcenter.tenant_embed_configs` with:
   - `tenant_id` text PK FK → tenants
   - `embed_key` text UK (public token, high entropy)
   - `enabled` bool default false
   - `allowed_origins` jsonb default `[]` (list of origin strings)
   - `default_agent_id` text nullable
   - audit: `created_at`, `updated_at`, `created_by`, `updated_by`
2. Unique index on `embed_key`.
3. Ensure-schema on server startup (same pattern as packages).
4. Optional seed: `demo` tenant row with `enabled=true` and empty allowlist (dev only) **or** lazy create on first tenant API access — document choice.
5. No breaking change to existing tenants without embed row.

## Verification

```bash
make infra-init && make restart

```
