# TASK-0140: Usage metering and billing/quota reconciliation contract

## Outcome

Define and implement the idempotent ClickHouse AI usage projection, versioned rate metadata, replay contract, and reporting-source boundaries for Sprint 31.

## Acceptance criteria

1. `ai_cost_usage_facts` has deterministic event identity, measurement state, source timestamps, rate version, currency, and audit/replacing semantics.
2. Duplicate delivery replaces one logical event and never doubles aggregate cost.
3. Text and voice meter inputs are provider-neutral and contain no prompt, response, transcript, audio, or customer identity.
4. Reporting joins paid orders, entitlements, call facts, and quota snapshots without mutating any source authority.
5. Missing usage or rate data remains unavailable/estimated and is covered by tests and replay notes.

## Review — PASS

The implementation adds a reporting-only `ai_cost_usage_facts` ReplacingMergeTree projection with deterministic event identity, source timestamps, rate metadata, measurement states, and `FINAL` aggregate reads. Metering is provider-neutral, excludes source content, preserves unavailable states when applicable rates are missing, and uses saturating cost arithmetic. Duplicate delivery is idempotent by fact identity. No payment, entitlement, quota, or customer interaction authority is mutated.

Verification: `make test`, server build, platform-admin checks/build, focused metering/ClickHouse tests, and `git diff --check` pass. Manual replay/fixture execution is listed in UAT-031 for tester follow-up.
