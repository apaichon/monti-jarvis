# TASK-0042: Callback Fulfillment

## Outcome

ChillPay callback completes checkout: order → `paid`, tenant gets package entitlement.

## Acceptance Criteria

1. `POST /api/callbacks/chillpay` matches `OrderNo` to `payment_orders`; `PaymentStatus=0` → `paid`, `paid_at`, `transaction_id`.
2. `PaymentStatus=2` → order `failed`; `1` → remains `pending`.
3. On `paid`: revoke prior `active` entitlement, assign new package (same transaction as order update).
4. Invalidate `monti_jarvis:entitlement:{tenant_id}` Redis cache.
5. Idempotent callback replay does not duplicate entitlement or error.
6. Mock pay dev route triggers same fulfillment path for E2E.
