# TASK-0098: Customer Portal Auth Gate and Workforce Picker

## Outcome

The customer portal guides customers through OTP sign-in before selecting an AI workforce when tenant policy requires it, while keeping the existing picker usable for optional-auth tenants.

## Acceptance criteria

1. Required-auth tenants show sign-in before the avatar/workforce picker is actionable.
2. Optional-auth tenants can continue without sign-in and can still sign in for account-aware support.
3. The popup picker lists only active tenant-assigned avatars and preserves selection after sign-in.
4. Signed-in customer identity and tenant are visible without exposing internal IDs unnecessarily.
5. Responsive layout keeps sign-in, current avatar, and picker access usable at 100% browser scale.
6. `npm run check` passes for `apps/customer-web`.

## Verification

```bash
cd apps/customer-web && npm run check
```
