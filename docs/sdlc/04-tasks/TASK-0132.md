# TASK-0132: Cross-tenant monitoring snapshot

## Outcome

Provide a request-time platform snapshot using bounded existing dependency probes, active tenant metadata, analytics freshness, and audit delivery health.

## Acceptance criteria

1. Shared dependencies use stable normalized statuses and bounded latency fields.
2. Tenant analytics reads are concurrent but bounded and never expose raw ClickHouse/provider errors.
3. Audit health contains only safe mode, status, counts, age, and last successful transfer.
4. No new persistent monitoring entity, cache, background poller, or customer-path dependency is introduced.

## Verification

- Implemented in `cmd/server/platform_monitoring.go` using `internal/observability`, existing store tenant listing, ClickHouse analytics, and audit writer health.
- Verified through authenticated endpoint smoke test with four active tenants and normalized dependency/audit states.
