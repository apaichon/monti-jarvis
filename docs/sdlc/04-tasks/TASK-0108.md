# TASK-0108: Ticket APIs

## Outcome

Customers can submit confirmed human-follow-up requests and tenant admins can list, inspect, update, and annotate tenant tickets.

## Acceptance criteria

1. Implement customer ticket creation with explicit confirmation, source conversation/call validation, existing auth-policy enforcement, and safe contact handling.
2. Implement tenant list/detail APIs with date, status, priority, category, avatar, customer, and assignee filters.
3. Implement validated lifecycle updates for status, priority, assignee, and internal note events.
4. Scope every read and write by tenant context; cross-tenant or missing ids return the same safe `404 not_found` response.
5. Repeated `Idempotency-Key` submission returns the original ticket without a duplicate open ticket.
6. Add unit/handler tests for authorization, validation, duplicate requests, invalid transitions, and tenant isolation.
