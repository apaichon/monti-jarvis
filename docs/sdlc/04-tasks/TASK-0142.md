# TASK-0142: Gemini AI usage instrumentation and platform usage dashboard

## Outcome

Capture normalized Gemini text/voice usage at the post-commit meter boundary and add the responsive platform billing usage surface.

## Acceptance criteria

1. Text provider usage metadata is captured when available; voice usage follows the observed/estimated/unavailable contract.
2. Metering failures cannot fail the customer call, voice relay, archive, or quota path.
3. `/admin/billing/usage` or the existing billing Usage tab shows paid value, reporting minutes, current quota state, AI coverage, cost state, and tenant rows.
4. Estimated cost is visually distinct from observed cost and no exact total is shown when coverage is unavailable.
5. Loading, empty, stale, unavailable, partial-failure, retry, session-expiry, and mobile states are implemented.

## Review — PASS

Gemini text usage metadata is captured after the conversation record commits and voice usage is emitted after the live relay completes. Meter writes are isolated from customer-call success paths. The platform billing Usage view includes filters, aggregate cards, coverage states, pricing metadata, tenant pagination, retry/session-expiry handling, and responsive layout; estimated or unavailable cost is not presented as an exact total.

Verification: full Go test, server build, platform-admin `svelte-check` (0 errors; 27 existing accessibility warnings outside this change), production build, and `git diff --check` pass.
