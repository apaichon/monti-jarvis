# TASK-0069: Tenant KM UI

## Outcome

Active tenants manage knowledge at `/tenant/km`.

## Acceptance Criteria

1. Route `/tenant/km` requires tenant session (same shell as billing/embed).
2. Agent selector (tabs or select) from `/api/tenant/km/agents`.
3. Document table: filename, scope, status, chunks, updated_at, actions (delete).
4. Upload control: file picker + scope select + submit; show success/error (incl. quota).
5. Reset agent button with confirm dialog.
6. Nav link **Knowledge** (or **KM**) in tenant shell.
7. API client under `apps/tenant-web/src/lib/api/km.ts` (or similar).
8. EN primary labels; short help: scopes map to caller topics (general/billing/technical).
