# TASK-0016: Tenant Entitlement Assign / Revoke + Read APIs

## Outcome

Platform operators assign packages to tenants; tenant admins read effective entitlement for their tenant.

## Acceptance Criteria

1. `POST /api/platform/tenants/{tenant_id}/entitlement` with `{package_id}` creates `active` entitlement; revokes prior active row; `404` for unknown tenant/package.
2. `DELETE /api/platform/tenants/{tenant_id}/entitlement` sets active entitlement to `revoked`; `404` when none active.
3. `GET /api/platform/tenants/{tenant_id}/entitlement` returns effective entitlement JSON (packages-spec §4 shape); `platform_admin` only.
4. `GET /api/entitlements/me` returns entitlement for JWT `tenant_id`; allowed for `tenant_admin` and `platform_admin`.
5. `customer` role receives `403` on entitlement routes.
6. Integration test: assign → tenant_admin reads `/api/entitlements/me` → revoke → `404` or empty documented default.
