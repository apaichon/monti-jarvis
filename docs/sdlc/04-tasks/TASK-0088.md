# TASK-0088: Customer and Import APIs

## Outcome

Tenant-admin REST APIs for customer CRUD/search, CSV dry-run/commit, import status, and domain-rule management.

## Acceptance criteria

1. All routes require active `tenant_admin`; tenant id comes only from JWT.
2. Implement customer list/create/get/update/deactivate endpoints defined in DES-0022.
3. Import accepts UTF-8 CSV, supports `dry_run`, caps file/row size, and returns row-level validation summaries.
4. Implement domain-rule list/create/update/delete with normalized unique domains.
5. Wrong-tenant resource ids return 404; malformed CSV/reference ids return 400; duplicates return 409 only when idempotent update is impossible.
6. Handler/store tests cover tenant isolation, dry-run no-write, and idempotent reimport.

## Verification

```bash
make test
curl -H "Authorization: Bearer $TOKEN" http://localhost:8091/api/tenant/customers
```

## Implementation evidence

- `cmd/server/tenant_customers.go` registers guarded CRUD, import, import-summary, and domain-rule handlers.
- `internal/customerimport` enforces UTF-8 CSV structure, known headers, validation, and row caps.
- Dry-run performs the same semantic validation as commit without customer writes.
- Commit stores valid customer upserts, memberships, and the import summary in one transaction.

## Review — PASS

Endpoint shapes, error codes, upload caps, dry-run behavior, tenant ownership, and atomic commit match DES-0022 and the API contract. Handler/parser tests and full Go tests pass.
