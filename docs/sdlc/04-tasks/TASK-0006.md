# TASK-0006: ClickHouse km_embeddings Schema + Search Client

## Outcome

Go package `internal/clickhouse` connects to ClickHouse, applies KM DDL, and exposes scoped vector search for RAG.

## Acceptance Criteria

1. `CLICKHOUSE_URL` added to `infra/.env.example` (default `http://localhost:8123`, database `monti_jarvis`).
2. DDL creates `km_embeddings` (tenant_id, km_version, chunk_id, km_scope, embedding `Array(Float32)`, content, updated_at) and `km_chunk_metadata` tables.
3. `UpsertEmbeddings` writes batch rows for a tenant/version.
4. `SearchScoped` accepts tenant_id, scopes `[]string`, query embedding, top-k; returns chunks sorted by cosine similarity.
5. `infra-check.sh` reports ClickHouse reachability when container or URL is configured.
6. Unit tests cover SQL builder / scope filter logic with mocked client.

## Verification

```bash
go test ./internal/clickhouse/...
./scripts/infra-check.sh
```
