# TASK-0094: Tenant Auth Settings UI

## Outcome

Tenant admin UI to enable customer authentication, review domain policy behavior, and understand production launch gate status.

## Acceptance criteria

1. Tenant admin can enable/disable customer auth from tenant settings or customer management.
2. UI exposes domain enforcement mode and links to existing domain rules.
3. UI communicates that production customer traffic requires quota/rate-limit isolation sign-off.
4. Unauthorized roles cannot read or mutate customer-auth settings.
5. Settings changes are reflected in API behavior and survive reload.

## Verification

```bash
npm --prefix apps/tenant-web run check
npm --prefix apps/tenant-web run build
```

## Implementation notes

- Added tenant settings panel for customer email OTP enablement, auth mode, allowed domains, OTP TTL, and session TTL.
- Settings use tenant-admin protected `/api/tenant/customer-auth/settings`.
