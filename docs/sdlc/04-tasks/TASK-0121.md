# TASK-0121: Tenant-Safe System Performance API

## Outcome

Expose a read-only monitoring snapshot for the authenticated active tenant administrator.

## Acceptance criteria

1. Add `GET /api/tenant/system-performance` behind the active tenant-admin guard.
2. Return overall status, checked time, normalized component results, and ClickHouse analytics freshness without tenant selection by request parameter.
3. Return safe `401`, `403`, and `503` responses for auth, tenant, and monitoring failures using the existing error shape.
4. Do not expose raw provider errors, credentials, customer data, transcripts, audio paths, or another tenant's identifiers.
5. Add handler tests for authorization, redaction, status aggregation, and dependency timeout behavior.

## Implementation Evidence

- Added `GET /api/tenant/system-performance` behind `RequireTenantAdminActive`.
- Tenant context is derived from the authenticated session; no tenant selector is accepted by the endpoint.
- Added ClickHouse analytics freshness mapping and safe normalized responses for missing auth and monitoring failures.
- Added handler tests for missing tenant context and provider-error redaction.

## Review - PASS

The handler follows the approved tenant-safe contract: it exposes only normalized component and analytics fields, does not serialize provider errors or tenant identifiers, and remains read-only.
