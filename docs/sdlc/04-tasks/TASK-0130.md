# TASK-0130: ClickHouse audit sink and query projection

## Outcome

Store audit events in a retry-safe ClickHouse projection and expose a bounded deduplicated read contract.

## Acceptance criteria

1. Runtime schema creation and migration are idempotent.
2. JSONEachRow inserts preserve metadata safely and use deterministic event IDs.
3. Uncertain delivery can be retried without duplicate logical results.
4. Query filters, pagination, and redacted output are bounded.

## Verification

- Added `audit_events` ReplacingMergeTree schema, JSONEachRow insert, `FINAL` query, date/filter validation, and cursor pagination.
- Added `scripts/migrations/003_audit_events_clickhouse.sql` and sink payload tests.
